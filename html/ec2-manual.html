Manually Deploy using AWS Developer Tools

Follow these steps to deploying a Java application on an EC2 instance -

  • EC2 - Create an EC2 Key Pair
  • S3 - Create an S3 bucket to store (version it)
  • IAM - Create an IAM role for CodeDeploy (see arn:aws:iam::114930652771:role/ABCMyNewCodeDeployRole) using the AWSCodeDeployRole Policy
  • CloudFormation - Launch a CFN stack (https://s3.amazonaws.com/stelligent-public/cloudformation-templates/github/labs/codebuild/CodeDeploy_SampleCF_Template.json)that launched an EC2 instance and installed the CodeDeploy agent on it
  • CodeBuild - Create a project using this as the source: https://github.com/stelligent/aws-codedeploy-sample-tomcat (select Artifacts packaging ZIP)
  • CodeDeploy - Manually create an application/DG
  • CodeDeploy - Deploy a new version and use the S3 bucket where the zip from the CodeBuild was stored
  • CodeDeploy/EC2 - Copy the public DNS and pasted into browser
